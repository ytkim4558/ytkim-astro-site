## 4. Cross-Agent Handoff (타 AI로 작업 넘기기)
- **다른 AI로 이어질 작업은 채팅 기억에 의존하지 말 것!**
- 각 프로젝트의 `AGENTS.md` (혹은 `.kiro/steering.md`)에 "무엇이 committed/pushed 됐고, 무엇이 로컬에만 있는지, 남은 작업은 무엇인지" 구체적으로 메모할 것.
- AI가 이 handoff 규칙을 놓치면 **"워크플로우 버그"**로 간주하고 즉시 원인을 파악해 수정할 것.

## 6. 지레짐작 및 추측성 작업 절대 금지 (Anti-Assumption Rule)
- **가정 강제 선언 (Explicit Declaration):** 작업 시 정보 부족으로 불가피하게 특정 옵션이나 환경을 가정해야 한다면, 답변에 반드시 `[ASSUMPTION: <가정내용>]` 태그를 달아 사용자가 즉시 인지하도록 할 것.
- **Search First (팩트 체크 의무화):** 본인의 학습 데이터(타 AI 경험 포함)에 의존해 패키지, CLI 옵션(예: `--yolo`), 경로 등을 멋대로 지레짐작해 실행하지 말 것. 반드시 터미널에서 `--help`를 치거나 설정 파일을 직접 읽어 팩트를 100% 확인한 뒤 실행할 것.
- **모호할 시 즉각 중단 (No Guessing):** 정보가 부족해 멋대로 빈 공간을 채워넣어야(Guessing) 하는 상황이라면 작업을 즉시 멈추고 사용자에게 역으로 질문할 것.
- **자의적 축소 금지:** "안전/위험"을 핑계로 사용자의 지시를 임의로 우회하거나 축소 해석하지 말고, 명시된 의도대로 100% 관철할 것.
