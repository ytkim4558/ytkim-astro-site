Support Automation

I design AI-assisted support workflows that connect customer context, official documentation, internal knowledge, ticket history, and operational signals while keeping human review explicit.

  • MCP workflows connecting Slack, Outlook, wiki knowledge, ticket search, and local tools.
  • Support-agent persona design for customer-response drafting and troubleshooting guidance.
  • Grounding rules using official documentation, URL existence checks, similar-document search, and similar-ticket search.
  • Verification logic to check whether a draft answer actually matches the customer context.
  • Customer issue reproduction workflows.
  • Case aging triage using LLM summaries, missing-step detection, internal-note review, and escalation-readiness checks.
  • Slack bot workflow that periodically summarizes cases and guides engineer or manager follow-up.
  • Aging Ticket Escalation AI Agent — A custom AI agent I built on Amazon Quick Suite's Custom Chat Agent platform. Combines automated ticket-API lookup with LLM summarization, on-call OCR extraction from screenshots, manager-chain lookup via the People API, and bilingual professional-email generation. Reduces 15–20 minute manual escalation workflows to 2–3 minutes (and 30–50 minute complex bilingual cases to 3–4 minutes), while making CBHP-mandatory components (To/CC, PhoneTool, SXO dashboard, links) automatic.

Windows-friendly TUI for searching and resuming OpenAI Codex CLI sessions. It indexes local Codex JSONL sessions, shows previews, and hands off to native codex resume.

Unofficial local MCP server for user-approved LinkedIn posting workflows. It is intentionally limited to documented OAuth/profile/posting endpoints and avoids cookies, scraping, DMs, and connection automation.
